Serverless Hands-on

Issac Henderson

Architecture

Here, we have a website that is designed in HTML, JavaScript & CSS, making it a completely static website. Deployed on Amazon S3, it utilizes the hosting of static website features, costing nearly zero cents.

Realizing the need for a contact page, I encountered a challenge in adding server-side logic to static websites. Static websites lack server-side languages like PHP, Java, etc., making it impossible to generate dynamic content, save information to a database, or send emails.

Serverless architectures, like AWS Lambda, offer a solution. They enable the addition of server-side logic to static websites, running scripts only when needed. For instance, AWS Lambda charges based on the number of requests and execution time, eliminating costs while waiting for user interactions.

Setup Amazon SES

As we know, Lambda functions will send emails, so we need to set up AWS SES.

Amazon SES requires you to verify your identities (the domains or email addresses you send emails) to confirm that you own them and prevent unauthorized use.

Follow the steps outlined in the Amazon SES user guide to verify your sender's email.

  1. Go to the Amazon SES dashboard and click on “Create Identity.”
  2. Make sure to select “Email” and enter your email ID from which you want to send the mail, and click on “Create Identity.”
  3. Now, your screen will be redirected to the SES dashboard, and you need to open your mail and verify that it belongs to you.
  4. Once you verify, refresh the Amazon SES dashboard and you will see the status changed to “Verified”.

Create Lambda Execution Role

Now, we need to create a Lambda execution role. That Lambda will use to get permission to access the Amazon SES service.

Let’s go to the IAM dashboard, select policy from the right panel and click “Create Policy.”

  1. Select JSON on the create policy page. Click next: Tag, then next: Review.

  2. Enter the policy name and click “Create Policy”.

Now let’s create the role. Select a role from the right panel and click on “create role.”

  1. On the Select trusted entity screen, select Lambda and click on “next.”
  2. Select the policy you created earlier and click Next.
  3. Enter the role name and click on “create role”. Remember this role as we will use it in the next step while creating the Lambda function.

Creating the API Gateway

Creating the API Gateway:

  1. Log in to the AWS console and select the API Gateway service. Go to the REST API and click on build.
  2. Select a new API, fill in your API name and click on “Create API.”
  3. Now the API is selected by default. Otherwise, you can also select this API from the dashboard. Now click on the “Actions” drop-down, and select “create resource.”
  4. Fill in the Resource Name. The path will be filled automatically for you, and click “Create Resource”.
  5. Select your newly-created resource and choose “create method.” Choose a POST.
  6. Here, you will choose our AWS Lambda Function. Search for the Lambda Function created earlier “contact” from the drop-down (Lambda Function) and save.
  7. You will get the message “Add the permission”. Click OK.
  8. After saving the form above, Click on the “action” menu and choose “deploy API.” Give the stage name “prod”, description and click on “Deploy”.
  9. Now get your Restful API URL from the “prod” tab, as shown in the screenshot below. We will use this URL on our “contact us” HTML page to send the request with all user information.
